## Overview

Pactus Blockchain [Version 1.16.0 (New Jersey)](https://github.com/pactus-project/pactus/releases/tag/v1.16.0)
is now available for download.
This release includes the following applications:

- [Pactus GUI](https://docs.pactus.org/get-started/pactus-gui/):
  The graphical user interface (GUI) for the Pactus node that is suitable
  for both beginners and experienced users.

- [Pactus Daemon](https://docs.pactus.org/get-started/pactus-daemon/):
  The command-line interface (CLI) for the Pactus node that is suitable for experienced users.

- [Pactus Shell](https://docs.pactus.org/tutorials/pactus-shell/):
  The command-line tool that allows users to interact with a Pactus node.

- [Pactus Wallet](https://docs.pactus.org/tutorials/pactus-wallet/):
  The wallet software that allows users to manage their wallet and send transactions
  without the need to sync the entire blockchain.

## Highlights

The most significant change in this release is the implementation of Block Reward Halving,
introduced through Pactus Improvement Proposal 55 ([PIP-55](https://pips.pactus.org/PIPs/pip-55)).
The block reward will follow a predictable reduction schedule:
starting at 1 PAC, it halves to 0.5 PAC at block 8,000,001,
to 0.25 PAC at block 24,000,001, and to 0.125 PAC at block 56,000,001.
This ensures a controlled and transparent monetary supply for the network over the long term.

Secp256k1 execution is now fully activated.
With the protocol upgrade to version 4, transactions can be sent to Secp256k1 addresses,
unlocking compatibility with the most widely adopted cryptographic curve in the blockchain industry—used
by Bitcoin and many hardware wallets.

The wallet now enforces password protection for vaults, closing a gap where unencrypted wallets could be accessed without
authentication. This is an important security improvement for all wallet users.

<div class="alert alert-warning block">
If you have any unencrypted wallets, we recommend setting a password before upgrading the node.
In the new release, unencrypted wallets will not be loaded and will return an error.
</div>

The GTK GUI adds system tray support with minimize-to-tray on close, a configurable snapshot download URL, and a more intuitive wallet
creation flow.

On the security front, several vulnerabilities have been addressed: a decompression bomb DoS vector (CWE-409) in P2P messages has been
mitigated with an 8 MB decompression limit, a zip-slip path traversal in wallet imports has been fixed, and an unbond authorization
bypass has been patched. Nil-pointer hardening across six message types further improves node resilience against malformed P2P traffic.

Under the hood, this release includes a new mock and fake test infrastructure that makes state, sandbox, and consensus tests more
reliable and easier to maintain, along with numerous consensus and sync improvements including double-proposal prevention, future
message discard in the consensus manager, and entropy-delayed block announcement broadcasts to reduce network spikes.

## Download

To start using Pactus blockchain, download the latest version from the [download](/download)
page and join the Mainnet.

## How to Upgrade

If you are running an older version, close it first.
Then uninstall the previous version and install the newer version.
If you are using the archived version, simply replace it with the new version.
